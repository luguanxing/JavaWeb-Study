4 QBC查询

# Hibernate检索策略
```
检索策略的概念
1 hibernate检索策略分为两类：
（1）立即查询：根据id查询，调用get方法，一调用get方法马上发送语句查询数据库
（2）延迟查询：根据id查询，还有load方法，调用load方法不会马上发送语句查询数据，只有需要对象非id值时候才会发送语句查询数据库
  类级别延迟：根据id查询返回实体类对象，调用load方法不会马上发送语句
  关联级别延迟：查询属主对象的从属对象是是否立即查询
  关联级别延迟操作: 在映射文件中进行配置实现

2 在set标签上使用属性
（1）fetch：值select（默认）
（2）lazy：值
- true：延迟（默认）
- false：不延迟
- extra：极其延迟
```
